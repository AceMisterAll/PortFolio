Stage dans l'entreprise K-line

J’ai effectué mon stage de deuxième année en entreprise au sein de la société « K-line». K-line est une entreprise située au Herbiers en Vendée. Elle fabrique des fenêtres, des baies coulissantes et des galandages, des portes d'entrée sur mesure en neuf et rénovation.

I. Le besoin de la mission n°1

Le besoin de ma première mission était la refonte de leur bandeau cookies et de que le bandeau soit responsive

II. Les ressources utilisées

: Pour réaliser le projet, j'ai utilisé
  • Un poste portable avec des logiciels afin de développer, tel que Visual Studio Code, Git Bash, Xampp qui permet de mettre en place un serveur web local
  • Un environnement de teste qui permet de tester avant de monté les modification en production
  • CMS Drupal, nous avons le possibilité de modifier le texte des pages grâce à l'interface graphique de Drupal mais aussi la possibilité de modifier les fichiers via un éditeur de code
  • GitLab
  • Figma, sur ce site je retrouvais les differentes maquette pour mes missions

III. Réallisation

Pour cette première mission voici les étapes que j'ai réaliser :
  • J'ai utiliser la partie graphique de Drupal afin de modifier le texte du bandeau cookie ainsi que le texte des boutons et des liens
  • Soit l’hébergement du système de contrôle était sur un serveur local dans l’entreprise

  • Par la suite j'ai modifier le css afin de faire correspondre la page à la maquette demander
  • Il m'a fallu utiliser du jQuery afin de déplacer des boutons car le bandeau cookies fait appel a une Library externe

  • C'est modification que j'ai du faire, j'ai du les copier dans le module du cookies et mettre a jour ces modifications sur GitLab. Afin que les différents site de Kline qui utilise se module cookies puisse avoir les dernières modifications
  • Puis sur les différents site qui utilisais se module, j'ai du mettre a jour le module afin de tester si les modifications
  • Jai du modifier le css afin que le bandeau cookies soit le même sur les différant site

Résultat de ladeuxième page

  • Pour la deuxième page, j'ai commencer par chercher les différence qu'il y avais entre la page actuel et la maquette
  • Ensuite, j'ai modifier le css afin la page ressemble à la maquette demander

Résultat de la deuxième page au format tablette

  • Cette page, j'ai du la rendre responsive avec un format tablette et mobile

Résultat de la deuxième page au format mobile

  • Pour le format mobile le design est différent des précédant. Car il m'a fallut crée une barre de navigation.
  • Pour crée cette barre, j'ai crée un fichier html.twig et j'ai ajouter les éléments souhaiter puis j'ai modifier le css afin qu'elle corresponde à la maquette
  • Puis, j'ai crée des fonctions en php afin de rendre les boutons de la barre de navigation fonctionnelles

    • Par exemple le chariot devais rediriger faire la page mon panier et le bouton avec le carton vers les commande à validé

    Pour le premier bouton a gauche celui-ci devais afficher une popup avec les information de contact

Pour la loupe celui-ci devais faire afficher l'interface de recherche avec le bouton qui li les qr code.
La partie du lecteur de Qr code était déja faite donc j'ai du importer cette partie qui était dans une autre branche git hub.
