Stage dans l'entreprise K-line

J’ai effectué mon stage de deuxième année en entreprise au sein de la société « K-line». K-line est une entreprise située aux Herbiers en Vendée. Elle fabrique des fenêtres, des baies coulissantes et des galandages, des portes d'entrée sur mesure en neuf et rénovation.

I. Le besoin de la mission n°1

Le besoin de ma première mission était la refonte de leur bandeau cookies et ce que le bandeau soit responsif

II. Les ressources utilisées

: Pour réaliser le projet, j'ai utilisé
  • Un poste portable avec des logiciels afin de développer, tel que Visual Studio Code, Git Bash, Xampp qui permet de mettre en place un serveur web local
  • Un environnement de test qui permet de tester avant de montée les modifications en production
  • CMS Drupal, nous avons la possibilité de modifier le texte des pages grâce à l'interface graphique de Drupal mais aussi la possibilité de modifier les fichiers via un éditeur de code
  • GitLab
  • Figma, sur ce site je retrouvais les différentes maquettes pour mes missions

III. Réallisation

Pour cette première mission voici les étapes que j'ai réalisé :
  • J'ai utilisé la partie graphique de Drupal afin de modifier le texte du bandeau cookie ainsi que le texte des boutons et des liens
  • Soit l’hébergement du système de contrôle était sur un serveur local dans l’entreprise

  • Par la suite j'ai modifier le css afin de faire correspondre la page à la maquette demander
  • Il m'a fallu utiliser du jquery afin de déplacer des boutons car le bandeau cookies fait appel à une library externe

  • C'est modification que j'ai dû faire, j'ai dû les copier dans le module des cookies et mettre à jour ces modifications sur Git lab. Afin que les différents sites de Kline qui utilise se modulent cookies puissent avoir les dernières modifications
  • Puis sur les différents sites qui utilisaient se module, j'ai dû mettre à jour le module afin de tester si les modifications
  • J'ai dû modifier le Css afin que le bandeau cookies soit le même sur les différent site

Le besoin de deuxième mission est de modifier le design de deux pages afin qu'elle corresponde à leur maquette donnée

V. Réallisation


Pour cette deuxième mission voici les étapes que j'ai réalisé :
  • Pour la première page j'ai modifié le design de quelque partie de la page afin de les faire correspondre avec le rendu souhaiter

Résultat de ladeuxième page

  • Pour la deuxième page, j'ai commencé par chercher les différences qu'il y avait entre la page actuelle et la maquette
  • Ensuite, j'ai modifié le Css afin que le page ressemble à la maquette demandée

Résultat de la deuxième page au format tablette

  • Cette page, j'ai dû la rendre responsive avec un format tablette et mobile

Résultat de la deuxième page au format mobile

  • Pour le format mobile le design est différent des précédent. Car il m'a fallu crée une barre de navigation
  • Pour créer cette barre, j'ai créé un fichier HTML.twig et j'ai ajouté les éléments souhaiter puis j'ai modifier le Css afin qu'elle corresponde à la maquette
  • Puis, j'ai créé des fonctions en Php afin de rendre les boutons de la barre de navigation fonctionnelle

    • Par exemple le chariot devait rediriger faire la page mon panier et le bouton avec le carton vers les commandes à valider

    Pour le premier bouton à gauche celui-ci devait afficher un pop-up avec les informations de contact

Pour la loupe celui-ci devait faire afficher l'interface de recherche avec le bouton qui li les Qr code.
La partie du lecteur de Qr code était déjà faite donc j'ai dû importer cette partie qui était dans une autre branche git hub.

VI. Documentation

J’ai développé la compétence « Travaille en mode projet ». En participant à des daily chaque jour, où l'on devait expliquer le travail réalisé de la veille.
J’ai aussi développé la compétence « Développer la présence en ligne de l’organisation » en participant à l’évolution d’un site Web exploitant les données de l’organisation (la refonte de plusieurs pages web ainsi que l'adaptation au format mobile.

VII. Bilan

Ce stage a été instructif pour moi, car j'ai pu voir comment était organiser le secteur informatique de l'entreprise, plus précisément celui du développement.
Sur ce stage de 7 semaines, j'ai pu apprendre de nouveau langage de programmation tel que le J'uery, il m'a aussi permet de me familiariser avec le Css et Git Hub. Ainsi que développer certaines compétences.
